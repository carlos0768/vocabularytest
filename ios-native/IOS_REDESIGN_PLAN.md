## Phase 3: Tinder機能（知ってる/知らない仕分け）
**新規ファイル:** `Features/Flashcard/TinderSortView.swift`, `TinderSortViewModel.swift`

### 仕組み
1. フラッシュカード画面のメニューから「仕分けモード」を選択
2. カードが1枚ずつ表示される（表面のみ）
3. **右スワイプ** = 知ってる（緑オーバーレイ + ✓）
4. **左スワイプ** = 知らない（赤オーバーレイ + ✗）
5. 全カード仕分け後、結果サマリーを表示
   - 知ってる: X語 / 知らない: Y語
   - 「知らない単語だけでクイズ」ボタン
   - 「知らない単語だけでフラッシュカード」ボタン

### UI
- カードは中央に1枚表示（Tinder風のスタック: 背面に次の2枚が少し見える）
- スワイプ中にカードが傾く + 色オーバーレイが出る
- スワイプ後、カードが画面外に飛んで次のカードが上がってくる
- 上部にカウンター（残りX枚）とプログレスバー

### アニメーション
- カードスタック: 奥のカードが `scale(0.95)` `scale(0.90)` で奥行き表現
- スワイプ: `rotation + translation + opacity` の組み合わせ
- 結果画面: 数字のカウントアップアニメーション

**変更量:** 2新規ファイル (~400行), FlashcardView.swift に遷移追加 (~20行)

## Phase 4: タイムアタック
**新規ファイル:** `Features/Quiz/TimeAttackView.swift`, `TimeAttackViewModel.swift`

### 仕組み
1. プロジェクト詳細画面 or ホーム画面から「タイムアタック」を選択
2. 制限時間を選択: 30秒 / 60秒 / 90秒
3. 4択クイズが次々と出題（既存のQuiz2と同じ問題形式）
4. 正解 → 即座に次の問題（0.3秒の緑フラッシュ）
5. 不正解 → 0.5秒の赤フラッシュ → 次の問題
6. タイマーが0になったら結果表示

### UI
- 上部: **大きなカウントダウンタイマー**（円形プログレス + 数字）
- 中央: 問題カード（英単語 + 4択）
- 下部: スコア表示（正解数 / 出題数）
- タイマーの色が残り時間で変化（青→黄→赤）

### 結果画面
- 正解数 / 出題数
- 1問あたり平均回答時間
- ベストスコア更新時に🎉エフェクト
- 「もう一度」「フラッシュカードで復習」ボタン

### アニメーション
- タイマー: 円形プログレスが減っていくアニメーション
- 正解/不正解: 画面フラッシュ + カード切り替え
- 結果: スコアのカウントアップ + confetti (シンプルな自前実装)

**変更量:** 2新規ファイル (~500行), 遷移追加 (~30行)

## 実装順序と理由

```
Phase 1 (ビルドエラー修正) → 他の全フェーズの前提
  ↓
Phase 2 (フラッシュカード強化) → 既存コードの改善、リスク低い
  ↓
Phase 3 (Tinder) → 新規画面、フラッシュカードの延長
  ↓
Phase 4 (タイムアタック) → 新規画面、Quiz2の延長
  ↓
Phase 5 (UIUX統合) → 全機能が揃った後にホーム画面を再構成
```
