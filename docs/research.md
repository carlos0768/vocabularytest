# MERKEN リサーチレポート

**調査日:** 2026年2月18日
**対象:** https://www.merken.jp/
**バージョン:** v1.0.0
**概要:** 高校生が個人開発したAIドリブンのカスタム単語帳Webアプリ

---

## 1. プロダクトの機能・特徴

### コアコンセプト
「手入力ゼロで単語帳を作成」-- 画像（写真）から英単語をAIで自動抽出し、日本語訳とともに単語帳に登録できるWebアプリ。従来の単語帳アプリが「手動入力」を前提としていたのに対し、MERKENは「撮影するだけ」というUXを実現している。

### 主要機能

#### (a) スキャン（画像からの単語抽出）
6つの抽出モードを搭載:
- **すべての単語** -- 写真内の英単語をすべて抽出
- **丸で囲んだ単語** -- マークした単語だけを抽出
- **ハイライト単語** -- 蛍光ペンで塗った単語だけを抽出
- **英検レベル** -- 指定した級の単語だけを抽出
- **熟語・イディオム** -- 句動詞や熟語だけを抽出
- **間違えた単語** -- テストの間違いだけを抽出

これは競合アプリにない独自機能であり、特に「丸で囲んだ単語」「ハイライト単語」モードは、教科書やプリントに直接マーキングして使うという学生の実際の学習行動に寄り添った設計。

#### (b) AI検索
英語・日本語で検索すると、関連する英単語を提案する機能。例: 「子犬」と入力すると puppy, dog, pet などが表示される。

#### (c) 単語帳管理
- 複数の単語帳を作成・管理
- 各単語帳に語数表示、カラフルなカード型UI
- 単語の手動追加・編集・削除・フラグ付け
- 検索・ソート（新しい順、単語が多い順、最近使った順）

#### (d) プロジェクト（コレクション）
複数の単語帳をグループ化して管理する機能。例: 「期末試験」プロジェクトに複数の単語帳をまとめる。

#### (e) 学習モード（4種類）
- **クイズ** -- 4択で意味を確認
- **カード** -- スワイプで復習（フラッシュカード形式）
- **例文クイズ** -- 例文で記憶を定着（Pro機能の可能性）
- **音声クイズ** -- 聞いて書く練習

#### (f) 統計・進捗管理
- **グローバル統計:** 今日のクイズ回答数、正答率
- **単語統計:** 習得済み / 復習中 / 未学習 の内訳、習得率
- **概要:** 単語帳数、総単語数、苦手単語数、間違えた単語数
- **単語帳別統計:** ドーナツチャートによる習得率表示、ステータス内訳

#### (g) 復習システム
ダッシュボードから「復習を始める（10問）」ボタンで横断的な復習が可能。単語帳をまたいだ復習に対応。

#### (h) オフライン対応
ダッシュボードに「オフライン」インジケーターがあり、Service Worker（sw.js）によるオフライン動作に対応。

#### (i) 共有機能
各単語帳に共有ボタンがあり、PWAのShare Target APIにも対応。

## 2. 技術スタック

プライバシーポリシーおよびソースコード解析から、以下の技術スタックが確認された。

### フロントエンド
| 技術 | 用途 |
|------|------|
| **Next.js** (App Router / RSC) | フレームワーク。`x-nextjs-prerender`, `vary: rsc, next-router-state-tree` ヘッダーで確認 |
| **React** | UIライブラリ（Next.js内蔵） |
| **Google Fonts** | フォント配信 |
| **PWA** | Service Worker (`sw.js`)、Web App Manifest、`standalone` display、`mobile-web-app-capable` |

### バックエンド・インフラ
| 技術 | 用途 |
|------|------|
| **Supabase** | 認証（Auth）・データベース（PostgreSQL）。`ryoyvpayoacgeqgoehgk.supabase.co` で確認 |
| **OpenAI API** | 画像解析・単語抽出。プライバシーポリシーで明記 |
| **Vercel** | ホスティング・CDN。`x-vercel-id: kix1::...` で確認（関西リージョン） |
| **Cloudflare** | CDN・Web Analytics（Cloudflare Insights） |
| **KOMOJU** | 決済処理（株式会社DEGICA運営） |

### データ保存アーキテクチャ
- **無料プラン:** IndexedDB（ブラウザローカル）にデータ保存。サーバーには送信されない
- **Proプラン:** Supabase（クラウド）に保存。デバイス間同期に対応

この「ローカルファースト + クラウド同期」のハイブリッドアーキテクチャは、コスト効率と段階的な価値提供の観点で非常に優れた設計判断。

### SEO関連
- OGP完備（og:title, og:description, og:image, Twitter Card）
- sitemap.xml あり（3ページ: /, /login, /signup）
- meta description, keywords 設定済み
- canonical URL未設定

## 3. ビジネスモデル

### 料金体系
| | 無料プラン | Proプラン |
|---|---|---|
| **月額** | 0円 | 500円/月 |
| **スキャン** | 制限あり（回数不明） | 無制限 |
| **単語数** | 制限あり（回数不明） | 無制限 |
| **データ保存** | ローカル（IndexedDB） | クラウド同期（Supabase） |
| **例文クイズ** | 不明 | 対応 |
| **音声クイズ** | 不明 | 対応 |

### 決済
- **KOMOJU** を利用（株式会社DEGICA運営のPCI DSS準拠決済サービス）
- 月額課金制
- 解約はいつでも可能、解約後も当月末まで利用可能
- 返金は原則不可
- 現時点では課金サブスクリプションではなく、課金操作不要との記載あり（テスト運用中の可能性）

### 収益性分析
- 月額500円は高校生にとって手が出しやすい価格帯
- OpenAI APIの画像解析コスト（GPT-4 Visionの場合1画像あたり数円〜数十円）を考慮すると、スキャン頻度が高いユーザーでは利益率が下がる可能性あり
- Supabaseの無料枠（500MB DB、1GB Storage）の範囲でスケールできるかが今後の課題

## 4. ユーザー体験（UI/UX）

### デザイン品質
- **評価: 高い。** 白を基調にした清潔感のあるデザイン、青のアクセントカラーで統一感がある
- ドット模様の背景が紙のノートを連想させ、学習アプリとしての世界観を構築
- カード型UIは直感的で、各単語帳が色分けされて視認性が高い
- アイコンデザインも統一されている

### モバイル対応
- **完全レスポンシブ対応**: デスクトップではサイドバー、モバイルではボトムタブバーに自動切替
- `viewport-fit=cover`, `user-scalable=no` でネイティブアプリに近い体験を実現
- `orientation: portrait` でポートレート固定

### PWA対応
- **完全PWA対応**: ホーム画面追加でネイティブアプリライクに動作
- Service Worker によるオフライン対応
- `standalone` ディスプレイモード
- Share Target API 対応（他アプリからテキストを共有して単語登録が可能）
- Apple Touch Icon 対応

### ナビゲーション
- 6つのセクション: ホーム、単語帳、プロジェクト、検索、統計、設定
- ダッシュボードに主要機能へのショートカット（スキャン、検索、コレクション、単語帳）
- 最近の単語帳のクイックアクセス

### テーマ
- ライト / ダーク / システム の3モード対応

### 改善点
- 404ページのデザインは適切だが、ナビゲーションリンクが「ホームへ戻る」「設定へ」のみで、やや限定的
- フッターにログアウト下のバージョン番号（v1.0.0）表示は良いプラクティス

## 5. 競合分析

### 主要競合との比較

| 特徴 | MERKEN | Anki | Quizlet | Monoxer | すごい暗記帳 |
|------|--------|------|---------|---------|------------|
| **AI画像抽出** | 6モード対応 | 非対応 | 非対応 | 非対応 | 対応（基本） |
| **マーク認識** | 丸囲み・ハイライト認識 | 非対応 | 非対応 | 非対応 | 非対応 |
| **英検レベル抽出** | 対応 | 非対応 | 非対応 | 非対応 | 非対応 |
| **間隔反復** | 基本対応 | 高度対応 | 基本対応 | AI最適化 | 対応 |
| **クイズ形式** | 4択・カード・例文・音声 | カード | 4択・筆記・マッチ等 | AI生成問題 | カード・4択・入力 |
| **価格** | 無料/500円月 | 無料(iOS 3,000円) | 無料/年額プラン | 学校向け/個人無料 | 無料/有料プラン |
| **プラットフォーム** | Web (PWA) | 全プラットフォーム | 全プラットフォーム | iOS/Android | iOS/Android |
| **オフライン** | 対応 | 対応 | 有料のみ | 対応 | 対応 |
| **SNS/共有** | 基本共有 | AnkiWeb | セット共有 | 学校内共有 | コミュニティ |
| **日本語特化** | 完全特化 | 多言語 | 多言語 | 日本語対応 | 日本語特化 |

### MERKENの差別化ポイント

1. **AI画像抽出の6モード** -- 最大の差別化要素。「丸で囲んだ単語だけ」「ハイライトした単語だけ」を認識できるアプリは他にない。これは教科書やプリントに実際にマーキングする学生の行動を深く理解した設計

2. **英検レベルフィルタリング** -- スキャン時に英検の級を指定して、そのレベルの単語だけを抽出できる。受験対策に直結する機能

3. **ローカルファースト設計** -- 無料プランでもIndexedDBでオフライン動作が可能。データがサーバーに送られないためプライバシー面でも優位

4. **PWAによるクロスプラットフォーム** -- ネイティブアプリ開発なしに、iOS/Android/PC全てに対応。個人開発者として効率的な技術選択

5. **低価格** -- 月額500円は学生にとって手頃。Ankiの3,000円一括やQuizletの年額プランより導入障壁が低い

### 競合に対する弱み

1. **Ankiの間隔反復アルゴリズム** には及ばない（MERKENは基本的な習得/復習/未学習の3段階管理）
2. **Quizletの巨大なコミュニティとセット共有エコシステム** がない
3. **Monoxerの学校・塾向け管理機能（教師ダッシュボード等）** がない
4. **ネイティブアプリがない** ため、プッシュ通知やウィジェットの制約がある
5. **英語以外の言語対応** が不明確（現状は英語-日本語特化）

## 6. SEO/マーケティング

### 現状の課題

#### 検索露出度: 非常に低い
- Google検索で「MERKEN 単語帳」「MERKEN アプリ」「merken.jp」で直接検索しても、MERKENのサイトがほぼ表示されない
- 主要な単語帳アプリ比較記事・ランキング記事にMERKENは掲載されていない
- App Store / Google Play にネイティブアプリが存在しないため、アプリストア経由の流入がゼロ

#### SNS存在感: ほぼなし
- Twitter/X上にMERKENに関する投稿・アカウントが確認できない
- 公式SNSアカウントの存在が不明

#### sitemapの課題
- sitemap.xmlに3ページ（/、/login、/signup）しか登録されていない
- ランディングページ（機能紹介・価格ページ等）が存在しない
- canonicalタグ未設定

#### コンテンツマーケティング
- ブログ・ヘルプセンター・チュートリアル等のコンテンツページが存在しない
- アプリ内の機能説明は充実しているが、検索エンジンからの流入を獲得するコンテンツがない

### OGP/ソーシャル共有
- OGPタグは適切に設定済み（og:title, og:description, og:image）
- Twitter Card は `summary` タイプ（`summary_large_image` への変更を推奨）
- OG画像はアプリアイコン（512x512）-- ソーシャル共有用の専用画像があるとより効果的

## 7. 強み・弱み・改善提案（SWOT分析）

### 強み (Strengths)
1. **独自のAI画像抽出技術** -- 6モードの抽出は業界唯一。特にマーク認識は他にない機能
2. **モダンな技術スタック** -- Next.js + Supabase + Vercel + OpenAI。高校生が個人でこの構成を選択・実装した技術力は非常に高い
3. **優れたUI/UX** -- プロフェッショナルレベルのデザイン品質
4. **完全PWA対応** -- オフライン動作、ホーム画面追加、Share Target
5. **ローカルファースト設計** -- 無料でもフル機能使える設計思想
6. **学生目線の設計** -- 実際の学習シーン（教科書にマーキングする行動）から逆算した機能
7. **プライバシー配慮** -- 画像データの非保存、ローカル保存オプション

### 弱み (Weaknesses)
1. **認知度の低さ** -- SEO/マーケティング施策がほぼなく、検索でヒットしない
2. **ランディングページ不在** -- 新規ユーザーに機能を訴求するページがない　←←←←←←←←←←←←←←←←←←
3. **ネイティブアプリ未提供** -- プッシュ通知、ウィジェットが使えない
4. **間隔反復の簡素さ** -- Ankiの高度なSRSアルゴリズムと比較すると基本的
5. **英語限定** -- 現状は英語-日本語のみ。多言語対応の余地
6. **コミュニティ機能の不足** -- 単語帳の共有・公開エコシステムがない
7. **無料/Proの違いの不透明さ** -- 無料プランの制限が明示されていない

### 機会 (Opportunities)
1. **EdTech市場の成長** -- 日本のEdTech市場は拡大中
2. **AI学習ツールへの関心** -- ChatGPT普及により、AI活用学習への関心が高い
3. **英検・受験対策需要** -- 英検レベルフィルタは受験生に直結するニーズ
4. **学校・塾への導入** -- B2B展開の可能性
5. **多言語展開** -- 中国語、韓国語など他言語への展開

### 脅威 (Threats)
1. **大手の参入** -- Quizlet、Duolingoが類似のAI機能を追加する可能性
2. **すごい暗記帳の成長** -- 類似コンセプトのアプリが既にストアに存在
3. **OpenAI APIコスト** -- 画像解析のコスト変動リスク
4. **個人開発のリソース制約** -- サポート、マーケティング、機能開発の同時進行が困難
