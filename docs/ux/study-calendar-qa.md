# 学習カレンダー UX QA 手順

## 目的
12週間固定カレンダーが、モバイル/デスクトップ双方で継続学習導線として機能するか確認する。

## 事前条件
- ローカル環境でアプリが起動していること。
- 少なくとも1日の学習履歴があるアカウント（またはローカル履歴）でログインできること。

## 確認項目
1. 統計ページを開くと、12週間のヒートマップが表示される。
2. 上部に「連続学習」「ベスト連続」「今週アクティブ」が表示される。
3. 「今日達成/今日未達」バッジが、今日の回答有無と一致する。
4. 任意のセルをタップ/クリックすると、日別詳細（問題数・正答率・状態）が即時更新される。
5. 日別詳細の「学習を続ける」から `/` へ遷移できる。
6. Proユーザーで統計ページを開いたあと、RPC取得結果でカレンダーが自然に更新される。
7. モバイル表示で横スクロール時にセルが押しづらくならない。

## 失敗時の切り分け
1. 表示日数が不足する場合:
   - `localStorage.scanvocab_activity_history` のデータ件数と日付範囲を確認。
   - `get_daily_stats_range` のレスポンス開始日が84日分になっているか確認。
2. タップしても詳細が変わらない場合:
   - セル `button` の `aria-label` と `selectedDate` state 更新を確認。
3. 週進捗が不正な場合:
   - `calculateCalendarSummary` の週開始（日曜基準）計算を確認。
