# 例文ジャンルパーソナライズ機能 コスト予測 (2026-06-10)

PR #173（`user_preferences.example_genres` によるジャンル指定例文生成）のコスト影響の試算。

## 結論（サマリ）

| 項目 | 影響 |
|------|------|
| AI呼び出し**回数** | **増えない**（既存の呼び出しにプロンプトを追記するだけ） |
| AI**入力トークン** | ジャンル設定ユーザの例文生成1回あたり **+100〜150トークン** |
| AI**出力トークン** | 変化なし |
| 直接コスト増 | **月あたり数十〜数百円規模**（下記モデル参照）。実質無視できる水準 |
| 二次的影響 | ジャンル設定ユーザの例文がマスター辞書に書き戻されなくなる → 長期的にマスターヒット率の伸びがわずかに鈍化（夜間バックフィルで大部分は相殺） |
| DBコスト | スキャン/生成APIごとに `user_preferences` へのSELECTが+1。無視できる |

## 前提

- 例文生成・クイズ生成はすべて **GPT-4o-mini**（`AI_CONFIG.defaults.openai`）。
  単価（試算時点・要再確認）: 入力 $0.15 / 1Mトークン、出力 $0.60 / 1Mトークン、$1 = 150円換算。
- ジャンル指示ブロックは日本語約135文字（ジャンル5件時）＝ **約100〜150トークン**。
  ジャンル未設定ユーザはブロック自体が挿入されないため **コスト増ゼロ**。
- 既存の単語1件あたり例文生成コール: 入力 約500トークン + 出力 約120トークン ≒ **0.022円/語**。

## 1. 直接コスト（プロンプト追記分）

### 呼び出し1回あたり

| 経路 | 追記トークン | 追加コスト/回 |
|------|------------|--------------|
| 単語ごとの例文生成（`/api/extract`, `/api/scan-jobs/process`） | +150 | $0.0000225 ≒ **0.0034円** |
| バッチ例文（`/api/generate-examples`、最大30語/回） | +150 | 同上（30語で按分されるため1語あたりはさらに小さい） |
| 誤答選択肢+例文（`/api/generate-quiz-distractors`、最大30語/回） | +150 | 同上 |

既存コール比では **入力トークン+約30%／コール総コスト+約15%**（単語ごと生成の場合）。
ただし絶対額が小さいため影響は軽微。

### 月間モデル

追加コスト = 月間スキャン数 × ジャンル設定ユーザ比率 × スキャンあたりAI例文生成語数 × 150トークン × $0.15/1M

| 月間スキャン | ジャンル設定率 | AI生成語数/スキャン | 追加コスト/月 |
|------------:|------------:|------------------:|-------------:|
| 10,000 | 30% | 10語 | 約 $0.68 ≒ **100円** |
| 50,000 | 30% | 10語 | 約 $3.4 ≒ **510円** |
| 100,000 | 50% | 10語 | 約 $11 ≒ **1,700円** |

※「AI生成語数/スキャン」は master-first 解決でマスター例文が当たらなかった語のみ。
マスターヒット率が上がるほどこの数は下がる。

## 2. 二次的コスト（マスター辞書への書き戻しスキップ）

ジャンル指定で生成した例文は個人向けのため、共有マスター（`lexicon_entries`）への
ベストエフォート書き戻しを **意図的にスキップ** している（`/api/extract`、
`/api/scan-jobs/process` server_cloud、`/api/generate-examples`）。

- 影響: ジャンル設定ユーザのスキャンがマスター例文の自然成長に寄与しなくなり、
  他ユーザのマスターヒット率の伸びがわずかに鈍る → その分のAI例文生成が長期的に増える。
- 緩和要因: 夜間の例文バックフィルcron（`20260324130000_nightly_example_backfill.sql`）が
  ユーザスキャンと無関係にマスター例文を埋めるため、**成長の主経路は維持される**。
- 定量化は困難だが、バックフィルが稼働している限り月間コストへの影響は
  上記の直接コストと同オーダー以下と見込む。

## 3. DB・レイテンシ

- `fetchExampleGenres()` による `user_preferences` SELECT が、スキャン1回・
  生成API1回ごとに+1クエリ。インデックス付きの単一行参照で、Supabaseの
  無料枠/Pro枠内では実質ゼロコスト。
- レイテンシ増は数十ms未満（例文生成のAIコール数秒に対して無視できる）。

## モニタリング

- 管理ダッシュボード `/api/ops/api-costs`（`api_cost_events`）で OpenAI 入力トークンの推移を確認。
- `feature_usage_daily` の `generate_examples` 使用回数が急増していないか確認。
- マスターヒット率は `scan_jobs.result` / extract ログの `masterHitCount` で追跡可能。

## 再試算が必要になる条件

- 例文生成モデルを GPT-4o-mini から変更したとき（`src/lib/ai/config.ts`）
- ジャンル上限（現在5件・30文字）を引き上げたとき
- 「マスター例文があってもジャンル設定ユーザにはAI再生成する」仕様に変えたとき
  （この場合はAI**呼び出し回数自体が増える**ため、影響は本書の試算より1桁以上大きくなる）
