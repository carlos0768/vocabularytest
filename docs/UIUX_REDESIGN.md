# ScanVocab UI/UX 設計案

## 📋 設計方針

### 基本原則
1. **手入力ゼロ** — アプリのコアバリューを最短距離で体験させる
2. **認知負荷を最小化** — 1画面で表示する情報を絞る
3. **達成感の可視化** — 学習の進捗をわかりやすく見せる
4. **導線の明確化** — 次にやるべきことが常にわかる
5. **モバイルファースト** — 片手操作で完結する設計

### 参考アプリ
- **Duolingo**: ゲーミフィケーション、達成感の演出
- **Anki**: 間隔反復の信頼性
- **Quizlet**: シンプルなカード学習
- **mikan**: 日本向け英語学習の参考

## 🏠 ホーム画面の再設計

### 現状の課題
1. プロジェクト一覧と詳細が同じ画面に混在
2. 「次にやるべきこと」が不明確
3. 学習の進捗が見えにくい
4. スキャンボタンの位置が目立たない

### 提案: ダッシュボード型ホーム

```
┌─────────────────────────────────┐
│  🔔  MERKEN          [アバター] │ ← ヘッダー
├─────────────────────────────────┤
│                                 │
│  ┌─────────────────────────┐   │
│  │  🔥 連続学習 5日目!     │   │ ← ストリーク表示
│  │  ████████░░ 80%         │   │
│  └─────────────────────────┘   │
│                                 │
│  ┌─────────────────────────┐   │
│  │  📚 今日の復習          │   │ ← 今日やるべきこと
│  │  12単語が復習時期です   │   │
│  │         [復習を始める]  │   │
│  └─────────────────────────┘   │
│                                 │
│  ┌─────────────────────────┐   │
│  │  🆕 最近のプロジェクト  │   │
│  │  • ノート P21-23 (25語) │   │
│  │  • 教科書 Ch.5 (18語)   │   │
│  │         [すべて見る →]  │   │

│                                 │
│                                 │
│           📷                    │ ← 中央フローティング
│        [スキャン]               │    スキャンボタン
│                                 │
├─────────────────────────────────┤
│  🏠    🔍    📊    ⚙️          │ ← ボトムナビ
└─────────────────────────────────┘
```

### ホーム画面の情報設計

| セクション | 内容 | 優先度 |
|-----------|------|--------|
| ストリーク | 連続学習日数、今日の達成率 | 高 |
| 今日の復習 | SM-2 で復習時期の単語数、開始ボタン | 高 |
| 最近のプロジェクト | 直近3件、タップで詳細へ | 中 |
| スキャンボタン | 中央フローティング、常に表示 | 最高 |

### 改善ポイント
1. **ストリーク表示**: 毎日の学習動機付け
2. **今日の復習**: 間隔反復に基づく「やるべきこと」の明示
3. **スキャンボタンの強調**: 中央フローティングで存在感アップ
4. **プロジェクト一覧は別ページへ**: ホームをシンプルに

### 現状
```
ボトムナビ: ホーム | 検索 | 統計 | 設定
```
