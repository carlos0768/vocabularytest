# MERKEN UI再設計ワイヤーフレーム案

## 方針
ホームは**学習ダッシュボード**、本棚は**管理と探索**に分離する。  
大量の単語帳をホームに並べず、ホームでは「今日やること」「続きから再開」「復習優先」のみを見せる。

---

## 案B：カード感を少し残す構成
画像の世界観を完全には捨てたくない場合。

### ホーム
- 今日の学習カード
- 続きから再開 1件
- 最近の単語帳 2列グリッドで4件まで
- 下部に「本棚へ」ボタン

### 本棚
- 上部に検索とフィルタ
- デフォルトは2列カード
- 件数が多いときは自動でリスト推奨表示

```text
本棚
[検索]
[言語▼][状態▼][並び替え▼]

┌────────────┐ ┌────────────┐
│ 鉄壁         │ │ CNN英単語    │
│ 48語         │ │ 31語         │
│ 復習待ち      │ │ 学習中        │
└────────────┘ └────────────┘
```

### 注意点
- 見た目は良いが、件数が増えたときに苦しくなる
- 画像が多いと分類しづらさが残る

## 案C：超実用型、学習効率最優先
受験・語学ガチ勢向け。少し無骨だが強い。

### ホーム
- 今日やるタスクだけ
- 復習待ち件数
- 新規学習件数
- 最後に触った単語帳

```text
MERKEN

今日やること
- 復習 12語
- 新規 4語
- 継続 7日

今すぐ再開
鉄壁 p.32-41
[復習する]

復習待ち一覧
- 英検準1級 Day 08
- 学校プリント Unit 5
- CNN英単語 3/12
```

### 本棚
- 完全にリストUI
- 検索、フィルタ、ソート特化

### 強み
- 大量データに最も強い
- 操作が速い
- 開発負荷も比較的低い

### 弱み
- 映えは弱い

---

## どれを選ぶべきか

### いちばんおすすめ
**案A**。  
理由は、見た目と運用のバランスがいちばんいいから。プロダクトとして長生きしやすい。

### こんなときはB
ブランド感やビジュアルの柔らかさを残したいとき。

### こんなときはC
まず学習体験を勝たせたい、MVPで早く固めたいとき。

## 具体的な修正指示

### 今のホームから削るもの
- 「すべての単語帳」セクション
- カード内のボタン2個並び
- 大きすぎる画像比率

### 今のホームに追加するもの
- 「続きから再開」
- 「復習待ち」縦リスト
- 「もっと見る」導線
- CTAボタン1つだけ

### 本棚に必要なもの
- 検索
- フィルタ
- 並び替え
- 表示切替
- ピン留め
- タグ

---

## UIルール

### カード
- ホームのカードは大カード1つ + 小カード数枚まで
- 一覧ではカードを軽くする
- ボタンはカード内に増やしすぎない

### 情報優先度
- 1位: 今日やるか
- 2位: 復習待ちか
- 3位: 最近使ったか
- 4位: 語数や作成日
- 5位: 画像

### ビジュアル
- 見出しサイズは少し抑える
- セクション右上に「もっと見る」
- 本棚では画像よりテキスト階層を主役にする
